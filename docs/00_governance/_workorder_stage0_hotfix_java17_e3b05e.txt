2-2. Gradle Toolchain 자동 해결 (설치 실패 시 대안)
**이게 빅테크 진짜 정석**: 개발자마다 JDK 설치 다를 수 있으므로 Gradle toolchain이 **자동 다운로드**하게 설정. `jvmToolchain(17)`만 쓰면 Gradle이 JDK 없으면 자동 설치.
# gradle.properties 추가
org.gradle.java.installations.auto-download=true
org.gradle.java.installations.auto-detect=true
이렇게 하면 JDK 17 수동 설치 없이도 빌드 성공. CI와 개발 환경 모두 일관.

    alias(libs.plugins.kotlin.jvm)
}

kotlin {
    jvmToolchain(17)
}

dependencies {
    implementation(libs.kotlin.stdlib)
    implementation(libs.kotlinx.coroutines.core)
    testImplementation(libs.junit.jupiter)
    testImplementation(libs.kotest.assertions)
    testImplementation(libs.kotlinx.coroutines.test)

tasks.test { useJUnitPlatform() }
3-3. 변경 요점
`java { sourceCompatibility / targetCompatibility }` 블록 **완전 삭제**
`kotlin { jvmToolchain(17) }` 블록 **신설**
Kotlin Gradle plugin 2.0+가 toolchain(17) 설정을 Java + Kotlin 양쪽에 자동 적용
결과: sourceCompatibility=17 / targetCompatibility=17 / jvmTarget=17 자동 일관화

4. 기존 서브프로젝트 일괄 승격 (전체 리포 통일)
4-1. 현황 파악
커서 작업 (Day 3.5-1):
리포 전체에서 `JavaVersion.VERSION_11` 사용하는 build.gradle.kts 전수 탐색
결과 파일 목록을 보고서에 포함
# 명령 예
grep -rn "VERSION_11" --include="*.gradle.kts" .
grep -rn "sourceCompatibility" --include="*.gradle.kts" .
4-2. 루트 build.gradle.kts에 subprojects 블록 추가
개별 모듈 모두 수정하는 대신 루트에서 일괄 적용 = 빅테크 정석
// 루트 build.gradle.kts

plugins {
    alias(libs.plugins.kotlin.jvm) apply false
    alias(libs.plugins.kotlin.android) apply false
    alias(libs.plugins.android.application) apply false
    alias(libs.plugins.android.library) apply false
}

subprojects {
    plugins.withId("org.jetbrains.kotlin.jvm") {
        extensions.configure<org.jetbrains.kotlin.gradle.dsl.KotlinJvmProjectExtension> {
            jvmToolchain(17)
        }
    }
    plugins.withId("org.jetbrains.kotlin.android") {
        extensions.configure<org.jetbrains.kotlin.gradle.dsl.KotlinAndroidProjectExtension> {
            jvmToolchain(17)
        }
    }
}
4-3. 개별 모듈 build.gradle.kts 정리
루트에서 toolchain을 일괄 적용하므로, 개별 모듈의 `java { sourceCompatibility / targetCompatibility }` 블록은 **전부 삭제**.
단, Android 모듈(android { compileOptions { ... } })의 Java 설정은 AGP 요구사항에 따라 VERSION_17로 명시
// 안드로이드 모듈 예 (app, feature/*)
android {
    compileOptions {
        sourceCompatibility = JavaVersion.VERSION_17
        targetCompatibility = JavaVersion.VERSION_17
    }
    // kotlinOptions { jvmTarget = "17" } 는 jvmToolchain이 자동 처리
}

5. gradle.properties 표준화
# gradle.properties 필수 항목

# JVM 메모리
org.gradle.jvmargs=-Xmx4g -Dfile.encoding=UTF-8

# Gradle Toolchain 자동 프로비저닝 (빅테크 정석)
org.gradle.java.installations.auto-download=true
org.gradle.java.installations.auto-detect=true

# 병렬 빌드 + 캐시
org.gradle.parallel=true
org.gradle.caching=true
org.gradle.configuration-cache=true

# Kotlin
kotlin.code.style=official
kotlin.incremental=true

# AndroidX (안드로이드 모듈 있으면)
android.useAndroidX=true
android.nonTransitiveRClass=true
핵심: `org.gradle.java.installations.auto-download=true`가 있으면 개발자가 JDK 17 없어도 Gradle이 자동 설치. **CI/개발 환경 일관성 정석.**
