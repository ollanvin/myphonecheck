# KISA 공식 피드 URL 확정 + 공공누리 라이선스 검토

**WO**: WO-V210-KISA-FEED-URL  
**발행**: 비전 (직접 발행) · standby 해제  
**코드 변경**: 0 (본 문서만)  
**조사 출처**: 공공데이터포털(`data.go.kr`), 한국인터넷진흥원(KISA) 공식 계열, KrCERT/CC(KISA 보호나라) 공개 페이지 · RSS 안내

---

## 1. 목적

MyPhoneCheck Architecture §30-4-3 **GovernmentPublic**(KR) 피드로 예정된 **KISA 계열 공개 데이터**에 대해:

1. **공공데이터포털에서 확인 가능한 데이터셋 고유 URL**(상세 페이지) 확정  
2. 메타데이터상 **이용허락범위**와 **공공누리 제1·2·3·4유형** 정합 검토  
3. KrCERT 보안공지 **RSS 공식 안내·목록 페이지** 정리(실제 `.xml` 피드는 안내 페이지 내 링크 확인)

---

## 2. 공공누리 유형 요약 (정합 기준)

문화체육관광부 「공공누리」 유형안내 기준(개념 요약):

| 유형 | 필수 조건(요지) |
|------|------------------|
| **제1유형** | **출처표시** |
| **제2유형** | 출처표시 + **상업적 이용금지** |
| **제3유형** | 출처표시 + **변경금지** |
| **제4유형** | 출처표시 + 상업적 이용금지 + 변경금지 |

**공공데이터포털** 각 파일데이터 메타의 「**이용허락범위**」는 위 공공누리 표기 또는 「**제한 없음**」 등으로 표시된다.

본 WO에서 상세 확인한 KISA 관련 파일데이터는 메타에 **「이용허락범위 제한 없음」** 및 동일 링크(`…/ugs/selectPortalPolicyView.do`)가 반복 표기되어 있다. 이는 통상 **공공데이터법 기반 개방**으로, **상업적 활용을 포함한 이용이 가능한 최대 개방 수준**으로 해석되는 표기다. 다만 **저작권·개인정보·별도 이용조건**은 각 데이터 설명란 및 포털 공통 정책을 따른다.

**MyPhoneCheck 제품 정합**: 유료 구독 앱(상업적 이용)으로서 **제2·제4유형 전용 데이터는 피드 원천으로 부적합**. 아래 확정 데이터셋은 포털 메타상 **「제한 없음」**이므로 **상업 앱 옵트인 피드와 정합 가능성이 높으나**, 실제 연동 직전에 **데이터셋 페이지를 재조회**하여 이용허락 변경 여부를 확인할 것.

## 4. KISA 공식 사이트 (직접 피드 URL)

KISA 대표 도메인(`www.kisa.or.kr`)은 웹 구조·메뉴 개편에 따라 **직접 다운로드 가능한 단일 정적 CSV 피드 URL**을 본 자동 조사에서 안정적으로 고정하지 못했다.

**권고**: 공식 **공개 데이터 다운로드**가 필요하면 **① 공공데이터포털 파일데이터**(위 §3)를 1차 정식 경로로 두고, **② KISA 사이트 내 「공공데이터」·「자료실」** 메뉴에서 동일 데이터의 원출처를 교차 확인한다.

## 5. KrCERT / 보호나라 — RSS·공지 URL

KrCERT·보호나라는 **CVE·보안공지·침해사고** 중심으로, §30의 **GovernmentPublic 피싱 URL CSV**와는 데이터 성격이 다를 수 있으나, **위협 인텔 연계·운영 공지**용으로 참조한다.

| 용도 | 확정 URL |
|------|-----------|
| RSS 이용 안내 | `https://www.krcert.or.kr/kr/subPage.do?menuNo=205121` |
| 보안공지 RSS 목록(KNVD 계열) | `https://knvd.krcert.or.kr/rssList.do` |
| 보호나라 보안공지 게시판 | `https://www.boho.or.kr/kr/bbs/list.do?bbsId=B0000133&menuNo=205020` |
| KrCERT 영문 보안공지 | `https://www.krcert.or.kr/en/bbs/list.do?bbsId=B0001041&menuNo=205083` |

**RSS/XML 직접 URL**: 플랫폼이 버전별로 상이할 수 있어, 위 **RSS 이용 안내·rssList** 페이지에서 구독 링크를 확인하는 방식이 공식 안내와 부합한다. 본 WO에서는 **단일 영구 `https://…/something.xml` 문자열을 확정하지 않음**(실무 시 해당 페이지에서 `.xml` 링크 확정).

## 6. 종합 권고 (Architecture §1·§30 정합)

1. **옵트인 피드 다운로드**(헌법 §1): 사용자 동의 후에만 서버가 아닌 **디바이스로 공공데이터포털 파일을 내려받아 캐시**하는 흐름과 정합.  
2. **출처 표시**: 공공누리 제1유형 및 「제한 없음」 데이터 공통으로 **제공기관(KISA)·공공데이터포털** 명시 권고.  
3. **개인정보**: 피싱 URL 목록은 공격 도메인 중심이나, **혼입 가능한 식별정보**가 없도록 포털 설명·법무 검토.  
4. **상업적 이용**: 본 보고서 확정 데이터셋은 메타 **「제한 없음」**. 다른 KISA 데이터셋을 추가할 때는 반드시 **제2·4유형** 여부를 스크린한다.
