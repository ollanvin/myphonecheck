# PART 2 — SPECIFIED (결정/명세 사항)

## 2.1 제품 아키텍처

### 2.1.1 One Core Engine, Six Surfaces

모든 Surface가 단일 코어 엔진(`:core:global-engine`)을 공유하는 구조.

| Surface | 모듈 | 기능 |
|---------|------|------|
| CallCheck | `:feature:call-check` | 의심 전화 실시간 검사 (CallScreeningService) |
| MessageCheck | `:feature:message-check` | 위험 링크 메시지 검사 (SmsBlockReceiver) |
| MicCheck | `:feature:privacy-check` | 마이크 무단 접근 감시 |
| CameraCheck | `:feature:privacy-check` | 카메라 무단 접근 감시 |
| PushCheck | `:feature:push-trash` | 푸시 알림 정크 필터 (NLS) |
| CardCheck | `:feature:card-check` | 카드 결제 문자 파싱 + 이상거래 탐지 |

### 2.1.2 4-Layer Data Model

| Layer | 정의 | 예시 |
|-------|------|------|
| L1 | OS 리소스 (읽기 전용) | CallLog, SMS Inbox, InstalledPackages |
| L2 | MyPhoneCheck 로컬 저장소 | Room DB (NKB), EncryptedSharedPreferences |
| L3 | 외부 공개 피드 캐시 (TTL 만료 후 폐기) | Abuse.ch, PhishTank, KISA |
| L4 | 외부 검색 (사용자 트리거 Only) | Google PSE, 일반 검색 엔진 |

### 2.1.3 Real-time Action Engine

- CallScreeningService: 전화 수신 시 50ms 내 판단
- SmsBlockReceiver: SMS 수신 시 즉시 분류
- NLS (NotificationListenerService): 푸시 알림 실시간 필터

### 2.1.4 모듈 구조 (18개 feature 모듈)

`:core:global-engine` — parsing (currency/phone/message/notification/timestamp), search (internal/external/public), sim-context, decision

Feature 모듈: call-check, message-check, card-check, push-trash, call-screening, initial-scan, settings, billing, tag-system, onboarding, search-enrichment, device-evidence, decision-engine, decision-ui, country-config, privacy-check, sms-block, call-intercept, message-intercept

### 2.1.5 SIM Context 체계

- `SimContext`: MCC/MNC → country/currency/phone format 자동 resolve
- UI 언어 3단 fallback: SIM default → Device System → English
- SIM 변경 시 사용자 3-옵션 선택 (새 컨텍스트 적용 / 기존 유지 / 초기화)

### 2.1.6 Global Parsing Engine

`:core:global-engine` 내 파싱 모듈:

- currency: ISO 4217 기반 통화 파싱
- phone: libphonenumber 기반 전화번호 파싱
- message: SMS 본문 패턴 추출
- notification: 푸시 알림 본문 파싱
- timestamp: 다국어 시간 표현 파싱

**핵심 원칙**: ICU/ISO 표준만 사용, 국가별 분기 코드 zero
